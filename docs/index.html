Why FakeCloud?

LocalStack replaced their open-source Community Edition with a proprietary image that requires an account and auth token. Their free tier is restricted to non-commercial use only.

FakeCloud is the fix. A fast, lightweight emulator for the AWS services you actually use in local dev. No login, no license key, no cloud account. Just run it and point your SDK at localhost.

Built for local dev

Focused scope, zero friction, actually works.

Simple

Single binary

One binary, one port, one command to run. Starts in under 100ms. Also available as a Docker image.

Open

No auth required

SigV4 signatures are parsed for routing but never validated. Use any dummy credentials.

Coverage

12 AWS services

S3, SQS, SNS, EventBridge, IAM/STS, SSM, DynamoDB, Lambda, Secrets Manager, CloudWatch Logs, KMS, CloudFormation. The services local dev actually needs.

Tested

Conformance tested

15,000+ auto-generated test variants validated against official AWS Smithy models, plus 280+ handwritten conformance tests and 213 native E2E tests.

Scheduling

EventBridge rules

Cron and rate-based scheduling that actually fires. Rules deliver to SQS and SNS targets on schedule.

Wired up

Cross-service delivery

S3 bucket notifications deliver to SNS/SQS. SNS fans out to SQS. EventBridge rules target SQS/SNS. S3 lifecycle runs in the background. Services talk to each other like they do on real AWS.

Quick Start

One command. No signup. No config file.

# Run with Docker
$ docker run -p 4566:4566 ghcr.io/faiscadev/fakecloud

# Or build from source
$ cargo install fakecloud
$ fakecloud

# Point any AWS SDK at localhost
$ aws --endpoint-url http://localhost:4566 sqs create-queue \
    --queue-name my-queue
